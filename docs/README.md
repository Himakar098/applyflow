# Documentation Guide

This repository keeps public-facing documentation small at the root and groups supporting material under `docs/`.

## Primary docs
- `README.md`: project overview, setup, verification, roadmap, and limitations
- `CONTRIBUTING.md`: contributor workflow and expectations
- `SECURITY.md`: private vulnerability reporting process
- `SUPPORT.md`: support and issue guidance
- `CHANGELOG.md`: release-oriented change tracking
- `AGENTS.md`: repository guidance for agentic coding runs

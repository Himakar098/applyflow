# Support

## When to open a GitHub issue
Open an issue for:
- reproducible bugs
- documentation problems
- feature requests
- setup issues that are specific to this repository

Please use the issue templates when possible.

## When not to open a public issue
Do not use public issues for:
- security vulnerabilities
- secrets, access tokens, or private credentials
- Firebase project details you should not expose publicly

For security-sensitive matters, follow [SECURITY.md](./SECURITY.md).

## What information helps
When asking for help, include:
- your operating system
- Node.js version
- whether you are running locally or on Vercel
- the command you ran
- the exact error message
- steps to reproduce

## Project support expectations
ApplyFlow is an early-stage open-source project. Support is best-effort rather than guaranteed real-time support.

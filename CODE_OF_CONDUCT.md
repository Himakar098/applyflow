# Code of Conduct

## Our standard
This project aims to be a respectful, practical, and constructive open-source space.

Participants are expected to:
- act professionally and respectfully
- assume good intent while still communicating clearly
- focus feedback on the work, not the person
- welcome contributors with different backgrounds and experience levels

Unacceptable behavior includes:
- harassment or abuse
- personal attacks
- discriminatory language or behavior
- doxxing or sharing private information without permission
- deliberately disruptive behavior
- bad-faith reporting or review conduct

## Scope
This Code of Conduct applies to:
- GitHub issues
- pull requests
- code reviews
- discussions around the project
- project-related communication channels maintained by the project owner

## Enforcement
Project maintainers may remove, edit, or reject comments, issues, commits, code, or other contributions that do not align with this Code of Conduct.

## Reporting
If you experience or witness unacceptable behavior, contact:
- `support.applyflow@gmail.com`

Please include as much context as possible. Reports will be reviewed and handled in good faith.

## Notes
This document is intentionally lightweight. As the project grows, it can be expanded or replaced with a more formal community standard if needed.
